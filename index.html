🔔 Webhook Patterns REFERENCE SCHEMA
In scope for AI CX Automation — designing this webhook payload pattern and wiring it through Make.com is the actual integration work. The schema below is the exact structure a Make.com HTTP module or Zapier webhook would send. Implement in production by pointing your Zendesk trigger at a Make.com webhook receiver.
# Zendesk → Make.com → NexusCS pattern # Step 1: Zendesk trigger fires on ticket create/update # Step 2: Make.com HTTP module sends to your endpoint POST https://hooks.nexuscs.io/ingest/YOUR_TOKEN { "source_tool": "zendesk", "subject": "{{ ticket.title }}", "priority": "{{ ticket.priority }}", "client_email": "{{ ticket.requester.email }}" }
